# 🤖 دليل تفعيل المساعد الذكي (Mistral) — خطوة بخطوة

هذا الدليل يشرح تفعيل ميزتين جديدتين أُضيفتا للمشروع:

1. **مساعد ذكي** يجيب الطالب من محتوى الدروس فعلياً (نص + عناوين فيديو/ملفات)، بزرّين عائمين:
   - 🤖 داخل صفحة كل درس → يجيب من هذا الدرس فقط.
   - 🤖 في الصفحة الرئيسية → يبحث عبر كل الدروس المنشورة بكل المراحل.
   - إن لم يجد إجابة داخل المصادر، يعرض رسالة ثابتة توجّه المستخدم للتواصل عبر الواتساب.
2. **جدولة نشر الدروس**: كل درس له الآن حقل "تاريخ ووقت النشر" في لوحة التحكم. الدرس بدون تاريخ = مسودة لا يراها الطلاب. الدرس بتاريخ مستقبلي = يظهر تلقائياً عند حلول ذلك الوقت.

> ⚠️ **مهم بخصوص طريقة العمل**: الاتصال بـ Mistral API يتم بالكامل من داخل **Supabase Edge Function** (على خوادم Supabase)، وليس من متصفح المستخدم. هذا يحمي مفتاح Mistral API من الظهور للعامة. لهذا السبب فتفعيل هذه الميزة يحتاج خطوة نشر إضافية عبر سطر الأوامر (لا يكفي رفع الملفات على Netlify فقط).

---

## الخطوة 1 — قاعدة البيانات

في **Supabase SQL Editor**، بعد أن تكون قد شغّلت `schema.sql` و `schema_admin.sql` مسبقاً، شغّل الآن:

```
schema_ai_assistant.sql
```

هذا الملف يضيف:
- عمود `published_at` لجدول `contents`
- تحديث سياسات RLS بحيث لا يرى الطالب إلا الدروس المنشورة فعلاً (المشرف يرى كل شيء)
- جدول `ai_chat_logs` لتسجيل أسئلة الطلاب (يساعدك على معرفة الأسئلة التي لم يستطع المساعد الإجابة عنها، أي فجوات المحتوى)

---

## الخطوة 2 — الحصول على مفتاح Mistral API

1. أنشئ حساباً على [console.mistral.ai](https://console.mistral.ai)
2. من قسم **API Keys** أنشئ مفتاحاً جديداً وانسخه (يظهر مرة واحدة فقط)

---

## الخطوة 3 — نشر الـ Edge Function

تحتاج جهازك (أو أي جهاز فيه Node.js) مرة واحدة فقط لتثبيت Supabase CLI ونشر الدالة. هذه الخطوة **لا يمكن إتمامها من داخل هذه المحادثة** لأنها تحتاج اتصالاً مباشراً بحسابك على Supabase.

```bash
# 1) تثبيت Supabase CLI (مرة واحدة فقط على جهازك)
npm install -g supabase

# 2) تسجيل الدخول
supabase login

# 3) اربط المشروع المحلي بمشروعك على Supabase
#    (SUPABASE_PROJECT_REF تجدها في: Settings → General → Reference ID)
cd رحلة-الحياة-الزوجية-PWA
supabase link --project-ref SUPABASE_PROJECT_REF

# 4) خزّن مفتاح Mistral كـ "سر" (Secret) — لا يظهر أبداً في الكود أو للمتصفح
supabase secrets set MISTRAL_API_KEY=ضع_مفتاحك_هنا

# (اختياري) لتغيير النموذج المستخدم، الافتراضي mistral-small-latest
supabase secrets set MISTRAL_MODEL=mistral-small-latest

# 5) انشر الدالة
supabase functions deploy ai-assistant
```

بعد نجاح النشر، ستظهر الدالة تحت: **Supabase Dashboard → Edge Functions → ai-assistant**

---

## الخطوة 4 — لا حاجة لأي تعديل في `config.js`

مكتبة `@supabase/supabase-js` المستخدمة أصلاً في المشروع تستدعي الدالة تلقائياً عبر:
```js
supabase.functions.invoke("ai-assistant", { body: {...} })
```
وتستخدم نفس `SUPABASE_URL` الموجود بالفعل في `js/config.js`، فلا حاجة لأي مفتاح API إضافي في كود الواجهة.

---

## الخطوة 5 — رفع الملفات الجديدة إلى GitHub ثم Netlify

نفس خطوات الرفع المعتادة الموجودة في `README.md`، فقط تأكد أن هذه الملفات الجديدة مرفوعة ضمن المستودع:
```
schema_ai_assistant.sql
css/assistant.css
js/assistant.js
supabase/functions/ai-assistant/index.ts
```
(مجلد `supabase/functions` لا يؤثر على نشر Netlify للواجهة، فهو يُستخدم فقط بواسطة أمر `supabase functions deploy`، لكن رفعه لـ GitHub مفيد لحفظ نسخة من كود الدالة).

```bash
git add .
git commit -m "إضافة المساعد الذكي وجدولة نشر الدروس"
git push
```

Netlify سينشر الواجهة تلقائياً عند كل push (إن كنت قد ربطت المستودع مسبقاً حسب `README.md`).

## اختبار الميزة

1. من لوحة التحكم → المحتوى، أضف/عدّل درساً واختر له **تاريخ نشر في الماضي أو الآن** حتى يظهر للطلاب.
2. افتح الدرس من حساب طالب عادي، اضغط 🤖 واسأل سؤالاً متعلقاً بمحتوى هذا الدرس تحديداً.
3. جرّب سؤالاً خارج نطاق الدرس (مثلاً عن موضوع غير موجود إطلاقاً) للتأكد من ظهور رسالة التوجيه للواتساب.
4. من الصفحة الرئيسية، جرّب 🤖 العام بسؤال يخص درساً في مرحلة مختلفة.

## استكشاف الأخطاء

| المشكلة | السبب المحتمل |
|---|---|
| "المساعد الذكي غير مُفعّل حالياً" | لم يُضبط `MISTRAL_API_KEY` كسر (secret) — أعد الخطوة 3 |
| "تعذّر الوصول إلى المساعد الذكي" | تحقق من رصيد حسابك على Mistral، أو راجع Logs الدالة من Supabase Dashboard → Edge Functions → Logs |
| الدرس لا يظهر للطلاب رغم حفظه | تأكد من ضبط "تاريخ ووقت النشر" بتاريخ حالي أو ماضٍ (وليس فارغاً أو مستقبلياً) |
| المساعد يجيب من خارج محتوى الدرس | راجع الفقرات (`content_sections`) المضافة فعلياً لهذا الدرس من لوحة التحكم، فالمساعد لا يقرأ إلا ما تم إدخاله هناك |
